大多数 Predoc Cover Letter 被忽略，背景不够强只是一方面，更常见的原因是写错了出发点。

PI 打开一封 Cover Letter，花在第一段的时间通常不超过 20 秒。他在找的是："这个学生有没有认真读过我的研究，他能不能马上上手"，热情与否是其次。如果第一段告诉 PI 的是"这份职位将为我提供宝贵的因果推断经验"，PI 会直接关掉。

这封信就这样被过滤了。背景弱不是主因，真正的问题是：你写的是你想要什么，没有说你能给什么。

---

## Cover Letter 和 PhD Personal Statement 是两件完全不同的事

很多学生用写 [PhD SOP](/guides/sop-guide) 的逻辑来写 Cover Letter。这个误区比你想象的普遍，而且它会直接导致申请失败。

两者之间的差异是结构性的：

| 维度 | Predoc Cover Letter | PhD Personal Statement |
|---|---|---|
| 读者 | 具体 PI，一个人 | 招生委员会，多人 |
| 核心问题 | "你能为我的项目做什么" | "你有潜力成为独立研究者吗" |
| 时间框架 | 短期，1–2 年，立即上手 | 长期，5 年以上，成长潜力 |
| 定制程度 | 必须针对具体 PI 高度定制 | 一版多用，微调 |
| 技能展示 | 具体工具和数据操作经验 | 研究问题意识和研究潜力 |

PI 招募的逻辑是：他需要一个现在就能帮项目推进的人，同时这个人未来申请 PhD 时能为他带来声誉回报。这和[如何判断 Predoc 导师是否值得合作](/guides/advisor-signal-decoding)里描述的 PI 视角一致。他在判断你能不能立即上手，"长期研究潜力"是 PhD 招生委员会关心的事。

SOP 里有价值的那套表述（"我从本科的某次经历开始对这个问题产生了长期关注"），在 Cover Letter 里没有任何用处。

## 以 PI 为中心，而不是以你为中心

这是 Cover Letter 最核心的一个原则，也是大多数人写错的地方。

看两段话的差别：

**❌ 以自己为中心（错误）：**
> "This position will provide me with valuable experience in causal inference and allow me to develop my skills in econometrics, which aligns with my long-term goal of pursuing a PhD."

**✅ 以 PI 为中心（正确）：**
> "Your recent work on peer effects in education uses a regression discontinuity design, and my experience building the administrative data pipeline for Professor X's education project at [机构] directly maps onto the data challenges in your 2024 working paper."

第一段给 PI 的信息是：这个学生想要从我这里获得什么。第二段给 PI 的信息是：这个学生具体能为我做什么。

写完每一段后，问自己：PI 读完这段，获得了什么关于"这个学生能为我做什么"的判断？这个逻辑和[套磁邮件](/guides/cold-email-guide)的核心原则完全相同。如果答案是"他了解到了这个学生很热情"，这段需要重写。

## 四段结构

Cover Letter 的标准长度是一页，通常四段。超过一页半需要压缩。

**第一段：开篇定位（1–2 句）**

说明申请的具体职位，加上与 PI 研究最关键的一个匹配点。不要写"我是一名热情的经济学学生"。

好的开篇应该让 PI 立刻知道：这个人认真读过我的研究，而且他能做相关的事情。

示例结构：
> "I am writing to apply for the predoctoral research position in your lab. My work on [具体方法/数据] in Professor X's project aligns directly with the empirical approach in your [论文名]."

**第二段：研究匹配（1–2 段）**

这是整封信里最需要花时间的部分。你需要具体说明：
- 你读了 PI 的哪篇论文或了解哪个项目
- 你自己的研究经历如何与 PI 的研究问题或方法产生实质交集
- 必须具体到论文的某个部分、某个识别策略、或某个数据挑战

"我对您的研究很感兴趣"不是研究匹配。"您在 2023 年的论文里用了 staggered DiD，我在 X 项目里处理过类似的 treatment timing 问题"才是研究匹配。

**第三段：技能证明（1 段）**

列举与该职位最相关的具体技能和数据经历。优先列最强的经历，其余作辅助。

常见错误：把最强的经历埋在第三、第四段，用大量平行陈述稀释了信号强度。正确做法是把最具竞争力的经历（比如[独立完成过 DiD 设计](/guides/independent-research-vs-assistant)、处理过特定数据库）放在最前面，其他的简短提及。

技能提及的格式：具体工具或方法 + 在哪个项目里用过 + 做了什么具体工作。"我熟悉 Stata"比"我使用 Stata 完成了 X 项目的面板数据构建，处理了跨年度的 merge 问题"弱得多。

**第四段：结尾（2–3 句）**

表达对该职位的期待，提供联系方式，不超过三句话。不需要在结尾再次回顾你的优势，那是前三段的工作。

## 定制化是唯一的出路

广撒网的策略在 Predoc 申请里几乎无效。

通用 Cover Letter 的回复率通常不到 5%。低回复率的根本原因，是申请信没有定制化，PI 一眼就能看出这封信同时发给了几十个人。

高定制化的少量申请（15–20 封），优于大量通用申请（100 封以上）。申请 Predoc 的整体策略，参考 [RA vs Predoc：真正的区别](/guides/ra-vs-predoc)。

定制化不只是把 PI 的名字换一下。真正的定制化是：你在第二段写的研究匹配内容，是专门针对这位 PI 的研究写的，换了另一个 PI 就完全失效。

## 最强经历前置原则

这是一个容易被忽略但影响很大的细节。

很多学生的 Cover Letter 结构是：背景介绍 → 经历 A → 经历 B → 最强经历 C → 技能列表。这样写，最关键的信息淹没在中间，PI 可能根本没读到。

正确的逻辑是：最强的经历放在前面，其余作辅助支撑。

如果你有过独立完成 DiD 设计的经历，或者处理过某个特定数据库（CRSP、PSID、Medicare 等），这应该在第一或第二段就出现，而不是埋在第三段的技能列表里。

---

## 附上可验证的能力证明

冷邮件和 Cover Letter 的区别在于，Cover Letter 通常随申请材料一起提交，而冷邮件往往是第一次接触。但两者都适用同一个原则：可验证的能力证明比任何自我描述都有说服力。

如果职位要求 Stata 或 Python，附上 GitHub 链接或 coding sample 比写"我熟练掌握 Stata"有效得多。PI 看到一段能跑通的数据清洗代码，比读十句"我具备扎实的数据处理能力"更能建立信任。

## 几类常见错误

**第一类：写了太多关于自己的"收益"。**

"这份工作将帮助我提升研究能力"、"我希望通过这段经历为 PhD 申请做准备"。这类表述让 PI 觉得他只是你的跳板。PI 不是在为你的 PhD 申请服务，他是在找一个能帮他推进项目的人。

**第二类：对 PI 研究的描述过于表面。**

"您的研究关注教育政策领域"不是研究匹配。这证明你大概搜索了一下 PI 的主页，但没有认真读他的论文。读最近 1-2 篇论文，找到一个具体的方法论细节或数据挑战，才能写出有说服力的匹配描述。

**第三类：把 Cover Letter 写成了 CV 的复述。**

Cover Letter 不是简历的文字版。简历已经列了你的经历，Cover Letter 要做的是解释这些经历为什么和 PI 的项目有关联，而不是重新列一遍。

**第四类：超过一页。**

PI 不会读超过一页的 Cover Letter。如果写超了，说明内容取舍还不够，压缩是唯一正确的做法。

## 如何准备写一封 Cover Letter

写一封 Cover Letter 之前，需要完成的准备工作：

**读 PI 的近期论文。** 至少读摘要和引言，最好读方法部分。你需要知道 PI 在用什么识别策略，面对的是什么数据挑战，在解答什么具体问题。

**找出你和 PI 项目的真实连接点。** 具体到操作层面，比如"我在 X 项目里处理过类似的面板数据结构"或"我熟悉 PI 用的 Y 识别方法"。如果找不到真实连接点，这个 PI 可能不是你目前阶段的合适目标。

**整理你的最强经历。** 想好你最能说明自己技术能力的 1-2 段经历，这是 Cover Letter 的核心素材。

**写第一稿，然后检查一遍：每段的主语是谁？** 如果大多数段落以"我"开头并描述你想要什么，需要重写。

## Cover Letter 和 SOP 的根本区别

很多人把 Cover Letter 和 SOP 混着写，导致两个材料都失去了各自的作用。理解这两者的区别，是写好 Cover Letter 的前提。

**SOP 的逻辑是：** 「我是谁，我为什么想研究 X，我有什么能力，这个项目为什么适合我。」核心主语是你，核心目的是让委员会认识你这个研究者。

**Cover Letter 的逻辑是：** 「你在招什么，我能做什么，为什么我和这个职位的需求是匹配的。」核心主语是这个职位的需求，核心目的是让导师相信你能胜任这个具体的工作。

这个区别在写作层面的体现是：SOP 可以用第一人称自由展开，Cover Letter 需要从职位需求出发，把自己的能力和需求对应起来，而不是单方面介绍自己。

## 针对不同职位的 Cover Letter 调整

Predoc 职位的要求差别很大，Cover Letter 必须针对每个职位做实质性调整。以下是几个需要根据职位定制的核心内容：

**技术技能的侧重。** 如果职位描述里强调「extensive data cleaning experience」，Cover Letter 里需要有具体的数据清理经历；如果强调「familiarity with causal inference methods」，需要说明你做过什么因果推断的实际工作。把职位描述里的关键词和你的实际经历对应起来。

**教授研究的呼应。** 每封 Cover Letter 里都需要有一段说明你读过这位教授的研究，以及你对他们当前项目有什么具体了解。这不需要很长（3-5 句话），但需要具体。泛泛的「您的研究对我非常启发」是无效的。

**你能贡献的具体价值。** 「我非常想学习您的研究方法」说的是你想从合作里得到什么；「我有 [具体技能]，可以帮助您的项目在 [具体方面] 更高效」说的是你能给这个项目带来什么。后者更有说服力。

**工作身份和时间安排。** 如果你有工作身份相关的限制或优势（比如 OPT、需要签证支持、可以远程），直接在 Cover Letter 里说清楚。关于工作授权问题的详细说明，参考[工作签证与身份](/guides/work-authorization-strategy)。

## Cover Letter 常见的结构框架

以下是一个可以参考的结构，每段功能明确：

**第一段（2-3 句）：** 说明你在申请什么职位（引用职位名称或职位发布来源），以及你为什么对这个教授的研究感兴趣。要具体，提一篇你读过的论文，或者他们当前项目的某个你了解的方面。

**第二段（3-4 句）：** 说明你最相关的技能和经历。针对这个职位，哪一段经历最能说明你能胜任？用一个具体的例子，说明你在相关任务上做过什么。

**第三段（2-3 句）：** 说明你能为这个项目带来什么具体价值，以及你的时间安排和工作意愿（全职、兼职、远程、起始时间）。

**第四段（1-2 句）：** 礼貌性收尾。「我附上了简历和 [Writing Sample/Coding Sample（如有要求）]，如有需要很乐意进一步说明。」不要用套话，简短即可。

整体长度控制在 300-400 词之间。太长的 Cover Letter 表明你不知道什么内容是重要的。

## FAQ

**Q：如果我对 PI 的研究了解不多，可以先发冷邮件问一问吗？**

不建议。在接触前不做功课，反而会给 PI 留下不认真的印象。真正缺乏背景知识的情况，说明这个 PI 的项目和你目前的技能栈匹配度可能不高，这时候更好的策略是先选更匹配的 PI 申请，等积累了更多相关经验后再接触这位 PI。

**Q：Cover Letter 需要每份都完全重写吗？**

第三段（技能证明）和第四段（结尾）可以用基础模板，但第二段（研究匹配）必须针对每位 PI 完全重写。第一段通常也需要调整。

**Q：没有相关领域经历，Cover Letter 怎么体现匹配？**

这时候要拆解"匹配"的维度。你可能没有做过完全相同领域的研究，但有可能：（1）用过相同的计量方法；（2）处理过类似结构的数据；（3）做过类似识别策略的复现。方法论层面的匹配往往比领域匹配更容易建立。

**Q：Coding Sample 应该放什么？**

放你做过的最能说明数据处理能力的代码。一段完整的、包含数据清洗和分析的 Stata 或 Python 脚本，附上简短注释说明在做什么。如果有 GitHub 仓库，直接附链接。代码不需要完美，需要能跑通、有注释、逻辑清晰。
